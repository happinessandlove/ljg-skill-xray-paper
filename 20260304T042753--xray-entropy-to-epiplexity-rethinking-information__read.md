# NAPKIN FORMULA

```
+----------------------------------------------------------+
|                                                          |
|   MDL_T(X) = S_T(X) + H_T(X)                            |
|                                                          |
|   Epiplexity = "可学结构" = 损失曲线下方面积 - 最终损失    |
|   Time-Bounded Entropy = "不可约噪声"                     |
|                                                          |
+----------------------------------------------------------+
```

在有限算力 T 下, 数据的总可压缩信息 = 可提取的结构性信息(epiplexity) + 无法进一步压缩的随机性(时间受限熵). 算力越大, 能提取的结构越多, 剩余的"噪声"越少.

# PROBLEM

**痛点定义**: Shannon 信息论和 Kolmogorov 复杂度无法刻画"有限算力的学习者能从数据中提取多少有用结构", 导致三个经典悖论无法解释:
1. 确定性变换不应增加信息, 但 AlphaZero 从简单规则学出复杂策略
2. 信息量与数据顺序无关, 但训练数据的排列顺序显著影响模型性能
3. 似然建模只是匹配分布, 但模型能提取比生成过程更丰富的结构

**前人困境**:
- Shannon 熵是"无限算力观察者"的度量, 对计算受限的神经网络不适用
- Kolmogorov 复杂度不可计算, 无法实际度量
- 现有信息论框架将伪随机数和真随机数视为等价(因为确定性变换不改变信息量), 但对有限算力的学习者而言, 伪随机数"看起来"就是随机的, 没有可学结构
- 没有一个理论框架能统一解释"数据价值"这件事: 什么数据对预训练更好? 为什么合成数据有用? 为什么数据顺序重要?

# INSIGHT

**核心直觉**: 信息的价值取决于"谁在看" -- 给你无限时间, 你能看穿一切加密; 但给你有限算力, 伪随机就是随机. 所以应该用"有限算力下能压缩出多少结构"来定义数据的信息量, 而非经典的 Shannon 熵.

**关键步骤**:
1. *引入计算预算 T 重新定义 MDL*: 将最小描述长度(MDL)限制在计算时间 T 内, 分解为两部分 -- 描述最优模型的"结构性位数"(epiplexity S_T) 和用该模型编码数据剩余的"随机位数"(时间受限熵 H_T). 这个分解让"可学结构"与"不可约噪声"有了严格区分
2. *连接密码学与学习论*: 用密码学安全伪随机数生成器(CSPRNG)做试金石 -- 伪随机序列的 Shannon 熵极低(种子很短), 但时间受限熵近乎最大(多项式时间算法无法区分它与真随机), epiplexity 几乎为零. 这优雅地证明了 epiplexity 确实捕获的是"可学结构"而非信息论意义上的"信息量"

# DELTA

**vs SOTA**:
- 相比 Shannon 信息论: 能解释确定性计算如何"创造"可学信息(元胞自动机从简单规则涌现出复杂结构, epiplexity 随模拟步数增长)
- 相比 Kolmogorov 复杂度: 可计算, 提出了两种实际估算方法(prequential 和 requential 编码)
- 相比 V-information 等已有计算受限信息度量: epiplexity 不需要预定义下游任务, 是数据本身的内禀度量
- 实验验证: epiplexity 与下游 OOD 泛化性能正相关; 文本数据的 epiplexity 高于图像数据, 解释了为什么语言预训练迁移性更好

**新拼图**: 为"以数据为中心的 AI"提供了理论基础 -- 数据的价值可以独立于特定任务来衡量, 合成数据可以通过计算创造新的可学结构, 数据排列顺序确实影响可学信息量. 这不是渐进式改进, 而是提出了一个新的信息论分支.

# CRITIQUE

**隐形假设**:
- 将计算预算 T 限制在神经网络参数化下的梯度下降优化, 实际的 "computationally bounded" 定义远比这窄 -- 不同优化器、架构、超参数都会改变 S_T 的值, epiplexity 其实是 (数据, 模型族, 优化算法) 三元组的函数, 而非数据本身的内禀属性
- prequential 编码估算 epiplexity 依赖"损失曲线下方面积"的直觉, 但没有严格证明该面积确实等于结构性位数, 只是"启发式地合理"
- 密码学论证(Theorem 9, 10)依赖单向函数假设(OWF), 这是未被证明的密码学假设(虽然广泛接受)
- 实验中文本 vs 图像的 epiplexity 对比未控制数据量和 tokenization 差异, 比较可能不够公平

**未解之谜**:
- Epiplexity 高是否真的意味着对下游任务有用? 论文承认"高 epiplexity 不保证与特定任务相关" -- 这是最大的实用性缺口
- 如何选择合适的计算预算 T? T 不同, epiplexity 的排序可能不同, 论文未给出指导
- 两种估算方法(prequential vs requential)虽然排序相关, 但数值差异大, 哪个更可靠?
- 能否用 epiplexity 指导实际的数据配比和课程学习? 论文仅停留在理论层面, 缺少端到端的实际应用验证

   ===============              ================

   "God's view"                 "Learner's view"
   (无限算力)                    (有限算力 T)

   CSPRNG seed: 128 bits        CSPRNG output:
   --> H = 128 bits              S_T ~ 0 (看不穿!)
   --> "信息很少"                 H_T ~ max (全是噪声)

   vs.                          vs.

   Game of Life rules:           Game of Life output:
   --> H ~ 几 bits               S_T >> 0 (涌现结构!)
